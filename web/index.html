Gestion documentaire 2.0

Ce document décrit ce qui est proposé pour structurer la gestion documentaire de l'OAQ. Il présente les choix qui ont été faits, les charges de travail estimées et les questions qui ne sont pas encore tranchées.

L'objectif n'est pas d'obtenir une approbation immédiate. C'est de donner suffisamment d'information pour que la direction générale forme son propre jugement et identifie ses propres questions avant la rencontre avec la présidence.

1

Vue d'ensemble

Ce qui est proposé

Une structure documentaire en six phases, déployée sur environ trois ans, qui définit où les documents vivent, comment ils sont nommés, comment ils circulent entre les espaces de travail et le C.A., et comment ils sont éventuellement publiés ou archivés. Le projet utilise les outils Microsoft 365 déjà en place et n'introduit aucun nouveau logiciel.

Périmètre du projet

Inclus

  • • Architecture des équipes Teams
  • • Nomenclature des fichiers
  • • Protocole de soumission au C.A.
  • • Gestion des canaux de projet
  • • Bibliothèque centrale SharePoint
  • • Espace documentaire du C.A.
  • • Registre des résolutions
  • • Migration des documents existants

Non inclus

  • • Courriels (Outlook)
  • • OneDrive personnels des employés
  • • Structure de publication sur Octave
  • • Outil d'IA — connexion conditionnée à la Phase 6
  • • Choix d'un système GED externe
  • • Numérisation des archives papier
2

Chaque phase peut être ouverte pour voir le détail. Les guides complets de chaque phase existent en document séparé — accessibles via l'assistant documentaire (section 7).

Prochaines étapes

La séquence proposée. Aucune étape n'est engagée tant que la rencontre avec la présidence n'a pas eu lieu.

Les guides détaillés de chaque phase et le guide TI complet sont disponibles séparément dans le dossier contenu/ du projet. Ils peuvent être consultés via l'assistant documentaire ci-dessous.

7

Assistant documentaire
